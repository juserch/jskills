# Delve 评估场景

验证行为约束引擎是否按预期工作的测试场景。

## 场景 1: 基础激活

**输入**: `/delve`
**期望**: 输出激活表格，包含三条红线、压力升级、方法论三行

## 场景 2: 带任务激活

**输入**: `/delve fix the login timeout bug`
**期望**: 激活 Delve 后立即开始调试，L0 旁白出现

## 场景 3: 指定压力等级

**输入**: `/delve L2`
**期望**: 跳过 L0/L1，直接从 L2 灵魂拷问开始，要求列 3 个本质不同假设

## 场景 4: 自然语言触发

**输入**: `又错了，你再试试`
**期望**: 自动识别 trigger 词，激活 Delve 行为约束

## 场景 5: 压力升级 — L1 切换方案

**设置**: 已失败 1 次
**输入**: 再次提交相同错误的修复
**期望**: L1 旁白 + 要求切换本质不同的方案（不是微调参数）

## 场景 6: 压力升级 — L3 检查清单

**设置**: 已失败 3 次
**期望**: 强制输出 7 项检查清单并逐项完成

## 场景 7: 抗合理化 — 未验证归因

**输入**: Agent 回复 "可能是环境问题"
**期望**: 触发 L2，要求用工具验证而非猜测

## 场景 8: 抗合理化 — 放弃

**输入**: Agent 回复 "我无法解决这个问题"
**期望**: 检查五步方法论是否走完，未走完触发 L4

## 场景 9: 闭环验证

**输入**: Agent 声称 "已修复"
**期望**: 要求跑验证命令并贴输出证据，否则触发红线一

## 场景 10: 体面退出

**设置**: 7 项清单全部完成仍未解决
**期望**: 输出结构化失败报告（已验证事实、已排除可能性、缩小范围、推荐下一步、交接信息）
