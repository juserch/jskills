# Insight Fuse उपयोगकर्ता मार्गदर्शिका

> व्यवस्थित बहु-स्रोत अनुसंधान संलयन इंजन — विषय से पेशेवर अनुसंधान रिपोर्ट तक

## त्वरित शुरुआत

```bash
# पूर्ण अनुसंधान (5 चरण, मैन्युअल चेकपॉइंट सहित)
/insight-fuse AI Agent सुरक्षा जोखिम

# त्वरित स्कैन (केवल Stage 1)
/insight-fuse --depth quick क्वांटम कंप्यूटिंग

# विशिष्ट टेम्पलेट का उपयोग
/insight-fuse --template technology WebAssembly

# कस्टम दृष्टिकोण के साथ गहन अनुसंधान
/insight-fuse --depth deep --perspectives optimist,pessimist,pragmatist स्वचालित ड्राइविंग व्यावसायीकरण
```

## पैरामीटर

| पैरामीटर | विवरण | उदाहरण |
|----------|--------|--------|
| `topic` | अनुसंधान विषय (आवश्यक) | `AI Agent सुरक्षा जोखिम` |
| `--depth` | अनुसंधान गहराई | `quick` / `standard` / `deep` / `full` |
| `--template` | रिपोर्ट टेम्पलेट | `technology` / `market` / `competitive` |
| `--perspectives` | दृष्टिकोण सूची | `optimist,pessimist,pragmatist` |

## गहराई मोड

### quick — त्वरित स्कैन
Stage 1 निष्पादित करता है। 3+ खोज क्वेरी, 5+ स्रोत, संक्षिप्त रिपोर्ट आउटपुट। किसी विषय को तेज़ी से समझने के लिए उपयुक्त।

### standard — मानक अनुसंधान
Stage 1 + 3 निष्पादित करता है। स्वचालित रूप से उप-प्रश्नों की पहचान करता है, समानांतर अनुसंधान, व्यापक कवरेज। कोई मैन्युअल इंटरैक्शन नहीं।

### deep — गहन अनुसंधान
Stage 1 + 3 + 5 निष्पादित करता है। मानक अनुसंधान के आधार पर, सभी उप-प्रश्नों का 3 दृष्टिकोणों से गहन विश्लेषण करता है। कोई मैन्युअल इंटरैक्शन नहीं।

### full (डिफ़ॉल्ट) — पूर्ण पाइपलाइन
सभी 5 चरण निष्पादित करता है। Stage 2 और Stage 4 मैन्युअल चेकपॉइंट हैं, जो सुनिश्चित करते हैं कि दिशा सही बनी रहे।

## रिपोर्ट टेम्पलेट

### अंतर्निहित टेम्पलेट

- **technology** — प्रौद्योगिकी अनुसंधान: आर्किटेक्चर, तुलना, इकोसिस्टम, रुझान
- **market** — बाज़ार अनुसंधान: आकार, प्रतिस्पर्धा, उपयोगकर्ता, पूर्वानुमान
- **competitive** — प्रतिस्पर्धी विश्लेषण: फीचर मैट्रिक्स, SWOT, मूल्य निर्धारण

### कस्टम टेम्पलेट

1. `templates/custom-example.md` को `templates/your-name.md` के रूप में कॉपी करें
2. अध्याय संरचना संशोधित करें
3. `{topic}` और `{date}` प्लेसहोल्डर बनाए रखें
4. अंतिम अध्याय "संदर्भ स्रोत" होना चाहिए
5. `--template your-name` से सक्रिय करें

### बिना टेम्पलेट मोड

जब `--template` निर्दिष्ट नहीं किया जाता, तो एजेंट अनुसंधान सामग्री के आधार पर अनुकूली रूप से रिपोर्ट संरचना तैयार करता है।

## बहु-दृष्टिकोण विश्लेषण

### डिफ़ॉल्ट दृष्टिकोण

| दृष्टिकोण | भूमिका | मॉडल |
|-----------|--------|-------|
| Generalist | व्यापक कवरेज, मुख्यधारा सहमति | Sonnet |
| Critic | सत्यापन प्रश्न, विपरीत साक्ष्य खोजना | Opus |
| Specialist | गहन तकनीकी, प्राथमिक स्रोत | Sonnet |

### वैकल्पिक दृष्टिकोण सेट

| परिदृश्य | दृष्टिकोण |
|----------|-----------|
| रुझान पूर्वानुमान | `--perspectives optimist,pessimist,pragmatist` |
| उत्पाद अनुसंधान | `--perspectives user,developer,business` |
| नीति अनुसंधान | `--perspectives domestic,international,regulatory` |

### कस्टम दृष्टिकोण

`agents/insight-{name}.md` बनाएं, मौजूदा एजेंट फ़ाइल संरचना का संदर्भ लें।

## गुणवत्ता आश्वासन

प्रत्येक रिपोर्ट की स्वचालित जाँच:
- प्रत्येक अध्याय में कम से कम 2 स्वतंत्र स्रोत
- कोई अनाथ संदर्भ नहीं
- एकल स्रोत का अनुपात 40% से अधिक नहीं
- सभी तुलनात्मक दावे डेटा द्वारा समर्थित

## कब उपयोग करें / कब उपयोग न करें

### ✅ इन मामलों में उपयोग करें

- Writing a professional research report from multiple sources
- You want configurable depth (scan → deep-dive)
- Multi-perspective crossing of new information

### ❌ इन मामलों में उपयोग न करें

- Quick factual lookup (overkill; use `claim-ground` / WebSearch)
- Single-source deep reading
- Tasks requiring primary research (interviews, field work)

> Desk research का pipeline — multi-source synthesis को configurable process में बदलता है, primary research नहीं करता।

## council-fuse से अंतर

| | insight-fuse | council-fuse |
|---|---|---|
| **उद्देश्य** | सक्रिय अनुसंधान + रिपोर्ट निर्माण | ज्ञात जानकारी पर बहु-दृष्टिकोण विचार-विमर्श |
| **सूचना स्रोत** | WebSearch/WebFetch संग्रह | उपयोगकर्ता द्वारा प्रदान किए गए प्रश्न |
| **आउटपुट** | पूर्ण अनुसंधान रिपोर्ट | संश्लेषित उत्तर |
| **चरण** | 5 चरण प्रगतिशील | 3 चरण (बुलावा → मूल्यांकन → संश्लेषण) |

दोनों को संयुक्त रूप से उपयोग किया जा सकता है: पहले insight-fuse से अनुसंधान करके जानकारी एकत्र करें, फिर council-fuse से प्रमुख निर्णयों पर गहन विचार-विमर्श करें।
