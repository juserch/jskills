# Руководство пользователя Tome Forge

> Начните за 5 минут — персональная база знаний с вики, компилируемой LLM

---

## Установка

### Claude Code (рекомендуется)

```bash
claude plugin add juserai/forge
```

### Универсальная установка одной командой

```
Fetch and follow https://raw.githubusercontent.com/juserai/forge/main/skills/tome-forge/SKILL.md
```

> **Нулевые зависимости** — Tome Forge не требует внешних сервисов, векторных баз данных или инфраструктуры RAG. Установите и работайте.

---

## Команды

| Команда | Что делает | Когда использовать |
|---------|-----------|-------------------|
| `/tome-forge init` | Инициализация базы знаний | При создании новой KB в любой директории |
| `/tome-forge capture [text]` | Быстрый захват заметки, ссылки или буфера обмена | Записать мысли, сохранить URL, вырезать контент |
| `/tome-forge capture clip` | Захват из системного буфера обмена | Быстрое сохранение скопированного содержимого |
| `/tome-forge ingest <path>` | Компиляция сырого материала в вики | После добавления статей, работ или заметок в `raw/` |
| `/tome-forge ingest <path> --dry-run` | Предпросмотр маршрутизации без записи | Проверка перед фиксацией изменений |
| `/tome-forge query <question>` | Поиск и синтез из вики | Поиск ответов по всей базе знаний |
| `/tome-forge lint` | Проверка здоровья структуры вики | Перед коммитами, периодическое обслуживание |
| `/tome-forge compile` | Пакетная компиляция всех новых сырых материалов | Догнать после добавления нескольких сырых файлов |

## Как это работает

На основе паттерна LLM Wiki Karpathy:

```
raw materials + LLM compilation = structured Markdown wiki
```

### Двухуровневая архитектура

| Уровень | Владелец | Назначение |
|---------|----------|-----------|
| `raw/` | Вы | Неизменяемые исходные материалы — работы, статьи, заметки, ссылки |
| `wiki/` | LLM | Скомпилированные, структурированные страницы Markdown с перекрёстными ссылками |

LLM читает ваши сырые материалы и компилирует их в хорошо структурированные страницы вики. Вы никогда не редактируете `wiki/` напрямую — вы добавляете сырые материалы и позволяете LLM поддерживать вики.

### Священный раздел

Каждая страница вики содержит раздел `## Моя дельта понимания`. Он **ваш** — LLM никогда не будет его изменять. Записывайте здесь свои личные озарения, несогласия или интуиции. Он сохраняется при всех перекомпиляциях.

## Обнаружение KB — Куда попадают мои данные?

Вы можете запустить `/tome-forge` из **любой директории**. Он автоматически находит нужную KB:

| Ситуация | Что происходит |
|----------|---------------|
| Текущая директория (или родительская) содержит `.tome-forge.json` | Использует эту KB |
| `.tome-forge.json` не найден выше | Использует `~/.tome-forge/` по умолчанию (создаётся автоматически при необходимости) |

Это значит, что вы можете захватывать заметки из любого проекта, не выполняя сначала `cd` — всё направляется в вашу единственную KB по умолчанию.

Хотите отдельные KB для каждого проекта? Используйте `init .` внутри директории проекта.

## Рабочий процесс

### 1. Инициализация

```
/tome-forge init                  # Create default KB at ~/.tome-forge/
/tome-forge init .                # Create KB in current directory
/tome-forge init ~/research-kb    # Create KB at a specific path
```

После инициализации директория KB выглядит так:

```
~/.tome-forge/               # (or wherever you initialized)
├── .tome-forge.json         # KB marker (auto-generated)
├── CLAUDE.md                # KB schema and rules
├── index.md                 # Wiki page index
├── .gitignore
├── logs/                    # Operation logs (monthly rotation)
│   └── 2026-04.md           # One file per month, never grows too large
├── raw/                     # Your source materials (immutable)
└── wiki/                    # LLM-compiled wiki (auto-maintained)
```

### 2. Захват

Из **любой директории** просто выполните:

```
/tome-forge capture "attention is fundamentally a soft dictionary lookup"
/tome-forge capture https://arxiv.org/abs/1706.03762
```

Быстрые захваты сохраняются в `raw/captures/{date}/`. Для более длинных материалов размещайте файлы непосредственно в `raw/papers/`, `raw/articles/` и т.д.

### 3. Загрузка

```
/tome-forge ingest raw/papers/attention-is-all-you-need.md
```

LLM читает сырой файл, направляет его на нужную страницу(ы) вики и объединяет новую информацию, сохраняя ваши личные заметки.

### 4. Запрос

```
/tome-forge query "what is the relationship between attention and transformers?"
```

Синтезирует ответ из вашей вики со ссылками на конкретные страницы. Сообщает, если информация отсутствует и какой сырой материал следует добавить.

### 5. Обслуживание

```
/tome-forge lint
/tome-forge compile
```

Lint проверяет структурную целостность. Compile пакетно загружает всё новое с момента последней компиляции.

## Формат страницы вики

Каждая страница вики следует строгому шаблону:

```yaml
---
domain: ai
maturity: growing        # draft | growing | stable | deprecated
last_compiled: 2026-04-15
source_refs:
  - raw/papers/attention.md
confidence: medium       # low | medium | high
compiled_by: claude-opus-4-6
---
```

Обязательные разделы:
- **Основная концепция** — Знания, поддерживаемые LLM
- **Моя дельта понимания** — Ваши личные озарения (LLM никогда не затрагивает)
- **Открытые вопросы** — Неотвеченные вопросы
- **Связи** — Ссылки на связанные страницы вики

## Рекомендуемый ритм

| Частота | Действие | Время |
|---------|---------|-------|
| **Ежедневно** | `capture` для мыслей, ссылок, буфера обмена | 2 мин |
| **Еженедельно** | `compile` для пакетной обработки материалов за неделю | 15-30 мин |
| **Ежемесячно** | `lint` + обзор разделов Моя дельта понимания | 30 мин |

**Избегайте загрузки в реальном времени.** Частая загрузка отдельных файлов фрагментирует связность вики. Еженедельная пакетная компиляция создаёт лучшие перекрёстные ссылки и более согласованные страницы.

## Дорожная карта масштабирования

| Фаза | Размер вики | Стратегия |
|------|------------|-----------|
| 1. Холодный старт (неделя 1-4) | < 30 страниц | Чтение полного контекста, маршрутизация через index.md |
| 2. Стабильное состояние (месяц 2-3) | 30-100 страниц | Шардирование по темам (wiki/ai/, wiki/systems/) |
| 3. Масштабирование (месяц 4-6) | 100-200 страниц | Запросы в рамках шарда, дополнение ripgrep |
| 4. Продвинутый (6+ месяцев) | 200+ страниц | Маршрутизация на основе эмбеддингов (не извлечение), инкрементальная компиляция |

## Известные риски

| Риск | Влияние | Смягчение |
|------|---------|----------|
| **Дрейф формулировок** | Многократная компиляция сглаживает личный стиль | `compiled_by` отслеживает модель; raw/ — источник истины; перекомпиляция из raw в любое время |
| **Потолок масштабирования** | Окно контекста ограничивает размер вики | Шардировать по домену; использовать индексную маршрутизацию; слой эмбеддингов при > 200 страницах |
| **Зависимость от поставщика** | Привязка к одному провайдеру LLM | Исходные материалы — чистый Markdown; сменить модель и перекомпилировать |
| **Повреждение дельты** | LLM перезаписывает личные озарения | Постзагрузочная проверка diff автоматически восстанавливает оригинальную дельту |

## Когда использовать / Когда НЕ использовать

### ✅ Используйте когда

- Building a personal knowledge base from scattered notes
- You want LLM-compiled wiki that preserves human insights
- Zero-infra solution (just Markdown + Git)

### ❌ Не используйте когда

- Team collaboration or real-time sync
- Ad-hoc notes (too structured — plain Markdown is fine)
- Transactional data (use a real database)

> Личная библиотека, скомпилированная LLM — сохраняет человеческие инсайты, рассчитана на одного, без real-time sync.

## FAQ

**В: Насколько большой может стать вики?**
О: До 50 страниц LLM читает всё. 50-200 страниц — использует индекс для навигации. Свыше 200 — рассмотрите шардирование по доменам.

**В: Можно ли редактировать страницы вики напрямую?**
О: Только раздел `## Моя дельта понимания`. Всё остальное будет перезаписано при следующей загрузке/компиляции.

**В: Нужна ли векторная база данных?**
О: Нет. Вики — это чистый Markdown. LLM читает файлы напрямую — без эмбеддингов, без RAG, без инфраструктуры.

**В: Как сделать резервную копию KB?**
О: Это всё файлы в git-репозитории. `git push` — и готово.

**В: Что если LLM допустит ошибку в вики?**
О: Добавьте исправление в `raw/` и загрузите повторно. Алгоритм слияния отдаёт предпочтение более авторитетным источникам. Или отметьте несогласия в своей Моя дельта понимания.
