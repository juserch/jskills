## Как это работает

Council Fuse сводит паттерн LLM Council Карпати в одну команду:

### Этап 1: Созыв

Три агента запускаются **параллельно**, каждый с уникальной перспективой:

| Агент | Роль | Модель | Сильная сторона |
|-------|------|--------|----------------|
| Универсал | Сбалансированный, практичный | Sonnet | Общепринятые лучшие практики |
| Критик | Состязательный, ищет недостатки | Opus | Крайние случаи, риски, слепые зоны |
| Специалист | Глубокая техническая детализация | Sonnet | Точность реализации |

Каждый агент отвечает **независимо** — они не видят ответы друг друга.

### Этап 2: Оценка

Председатель (основной агент) анонимизирует все ответы как Ответ A/B/C, затем оценивает каждый по 4 параметрам (0-10):

- **Корректность** — фактическая точность, логическая обоснованность
- **Полнота** — охват всех аспектов
- **Практичность** — применимость, пригодность в реальных условиях
- **Ясность** — структура, читаемость

### Этап 3: Синтез

Ответ с наивысшей оценкой становится основой. Уникальные выводы из других ответов интегрируются. Обоснованные возражения критика сохраняются как оговорки.

## Примеры использования

### Архитектурные решения

```
/council-fuse Should we use microservices or a monolith for our 10-person team?
```

Универсал предоставляет сбалансированные компромиссы, критик оспаривает ажиотаж вокруг микросервисов, а специалист детализирует паттерны миграции. Синтез даёт условную рекомендацию.

### Выбор технологий

```
/council-fuse Redis vs PostgreSQL for our job queue
```

Три различных ракурса гарантируют, что вы не упустите эксплуатационные проблемы (критик), детали реализации (специалист) или прагматичный выбор по умолчанию (универсал).

### Ревью кода

```
/council-fuse Is this error handling pattern correct? <paste code>
```

Получите в одном проходе: валидацию мейнстримом, состязательный анализ крайних случаев и глубокую техническую проверку.

## Структура вывода

Каждое обсуждение совета выдаёт:

1. **Матрица оценок** — прозрачная оценка всех трёх перспектив
2. **Анализ консенсуса** — в чём они согласны и в чём расходятся
3. **Синтезированный ответ** — объединённый лучший ответ
4. **Мнение меньшинства** — обоснованные альтернативные точки зрения, достойные внимания

## Настройка

### Изменение перспектив

Отредактируйте `agents/*.md`, чтобы определить собственных членов совета. Альтернативные триады:

- Оптимист / Пессимист / Прагматик
- Архитектор / Реализатор / Тестировщик
- Защитник пользователя / Разработчик / Эксперт по безопасности

### Изменение моделей

Отредактируйте поле `model:` в каждом файле агента:

- `model: haiku` — экономичные советы
- `model: opus` — тяжеловесные советы для критических решений

## Часто задаваемые вопросы

**В: Это стоит в 3 раза больше токенов?**
О: Да, примерно. Три независимых ответа плюс синтез. Используйте для решений, которые оправдывают инвестицию.

**В: Могу ли я добавить больше членов совета?**
О: Фреймворк это поддерживает — добавьте ещё один файл `agents/*.md` и обновите рабочий процесс в SKILL.md. Однако 3 — оптимальный баланс между стоимостью и разнообразием.

**В: Что если один из агентов выйдет из строя?**
О: Председатель ставит этому члену оценку 0 и синтезирует из оставшихся ответов. Плавная деградация, без сбоев.
