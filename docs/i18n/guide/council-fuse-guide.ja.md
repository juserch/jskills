## 仕組み

Council Fuse は Karpathy の LLM Council パターンを単一コマンドに凝縮します：

### ステージ1：招集

3つのエージェントがそれぞれ異なる視点で**並列に**生成されます：

| エージェント | 役割 | モデル | 強み |
|------------|------|--------|------|
| ジェネラリスト | バランス型、実践的 | Sonnet | 主流のベストプラクティス |
| クリティック | 対立的、欠陥を発見 | Opus | エッジケース、リスク、盲点 |
| スペシャリスト | 深い技術的詳細 | Sonnet | 実装の精度 |

各エージェントは**独立して**回答します — 他のエージェントの回答は見えません。

### ステージ2：採点

議長（メインエージェント）がすべての回答を回答A/B/Cとして匿名化し、4つの次元（0-10）で採点します：

- **正確性** — 事実の正確さ、論理的健全性
- **完全性** — すべての側面のカバー範囲
- **実用性** — 実行可能性、現実世界での適用性
- **明瞭性** — 構造、読みやすさ

### ステージ3：統合

最高得点の回答が骨格となります。他の回答からのユニークな洞察が統合されます。クリティックの妥当な異議はただし書きとして保持されます。

## ユースケース

### アーキテクチャの意思決定

```
/council-fuse Should we use microservices or a monolith for our 10-person team?
```

ジェネラリストがバランスの取れたトレードオフを提供し、クリティックがマイクロサービスの誇大広告に異議を唱え、スペシャリストが移行パターンを詳述します。統合により条件付きの推奨が得られます。

### 技術選定

```
/council-fuse Redis vs PostgreSQL for our job queue
```

3つの異なる角度により、運用上の懸念（クリティック）、実装の詳細（スペシャリスト）、実用的なデフォルト（ジェネラリスト）を見逃しません。

### コードレビュー

```
/council-fuse Is this error handling pattern correct? <paste code>
```

主流の検証、対立的なエッジケース分析、深い技術的検証を一度に取得できます。

## 出力構造

各評議会審議は以下を生成します：

1. **スコアマトリックス** — 3つの視点すべての透明な採点
2. **合意分析** — 一致点と不一致点
3. **統合回答** — 融合された最良の回答
4. **少数意見** — 注目に値する妥当な反対意見

---

## カスタマイズ

### 視点の変更

`agents/*.md` を編集してカスタム評議会メンバーを定義します。代替トライアド：

- 楽観主義者 / 悲観主義者 / 現実主義者
- アーキテクト / 実装者 / テスター
- ユーザー擁護者 / 開発者 / セキュリティ専門家

### モデルの変更

各エージェントファイルの `model:` フィールドを編集します：

- `model: haiku` — コスト効率の良い評議会
- `model: opus` — 重要な意思決定のためのオール・ヘビー級

## 使用場面 / 不向きな場面

### ✅ 使うべきとき

- Architectural decisions needing multiple perspectives
- You suspect confirmation bias in your current answer
- You want dissenting views preserved, not erased

### ❌ 使うべきでないとき

- Pure factual queries (use `claim-ground` instead)
- Creative writing (synthesis flattens style)
- Speed-critical decisions (3 agents run sequentially — not fast)

> 訓練知識ベースの討論エンジン — 単一視点の盲点を露わにするが、結論は訓練知識に制約される。

完全な境界分析: [references/scope-boundaries.md](../../../skills/council-fuse/references/scope-boundaries.md)

## よくある質問

**Q：トークンが3倍かかりますか？**
A：はい、おおよそそうです。3つの独立した回答と統合処理があります。投資に見合う意思決定に使用してください。

**Q：評議会メンバーを追加できますか？**
A：フレームワークはサポートしています — `agents/*.md` ファイルを追加し、SKILL.md のワークフローを更新してください。ただし、コストと多様性のバランスでは3が最適です。

**Q：エージェントが失敗した場合はどうなりますか？**
A：議長はそのメンバーに0点を付け、残りの回答から統合します。優雅な劣化であり、クラッシュしません。
