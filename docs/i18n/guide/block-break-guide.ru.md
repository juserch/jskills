# Block Break Руководство пользователя

> Начните за 5 минут — заставьте вашего ИИ-агента исчерпать все подходы

---

## Установка

### Claude Code (рекомендуется)

```bash
claude plugin add juserai/forge
```

### Универсальная однострочная установка

```
Fetch and follow https://raw.githubusercontent.com/juserai/forge/main/skills/block-break/SKILL.md
```

> **Ноль зависимостей** — Block Break не требует внешних сервисов или API. Установите и начинайте.

## Команды

| Команда | Что делает | Когда использовать |
|---------|-----------|-------------------|
| `/block-break` | Активировать движок Block Break | Повседневные задачи, отладка |
| `/block-break L2` | Начать с определённого уровня давления | После известных множественных неудач |

## Сценарии использования

### ИИ не смог исправить баг после 3 попыток

Введите `/block-break` или скажите `try harder` — автоматически входит в режим эскалации давления.

### ИИ говорит «вероятно, проблема окружения» и останавливается

Красная линия «Основано на фактах» Block Break принуждает к проверке инструментами. Непроверенное приписывание причин = перекладывание вины → срабатывает L2.

### ИИ говорит «предлагаю вам сделать это вручную»

Срабатывает блок «Мышление владельца»: если не ты, то кто? Прямо L3 Оценка производительности.

### ИИ говорит «исправлено», но не показывает доказательств проверки

Нарушает красную линию «Замкнутый цикл». Завершение без вывода = самообман → принуждает к командам проверки с доказательствами.

## Основные механизмы

### 3 Красные линии

| Красная линия | Правило | Последствие нарушения |
|---------------|---------|----------------------|
| Замкнутый цикл | Необходимо выполнить команды проверки и показать вывод перед заявлением о завершении | Срабатывает L2 |
| Основано на фактах | Необходимо проверить инструментами перед приписыванием причин | Срабатывает L2 |
| Исчерпать всё | Необходимо завершить 5-шаговую методологию перед фразой «не могу решить» | Прямо L4 |

### Эскалация давления (L0 → L4)

| Неудачи | Уровень | Боковая панель | Принудительное действие |
|---------|---------|----------------|----------------------|
| 1-я | **L0 Доверие** | > Мы тебе доверяем. Не усложняй. | Нормальное выполнение |
| 2-я | **L1 Разочарование** | > Соседняя команда справилась с первой попытки. | Переключиться на принципиально другой подход |
| 3-я | **L2 Допрос** | > В чём корневая причина? | Поиск + чтение исходного кода + перечислить 3 разные гипотезы |
| 4-я | **L3 Оценка производительности** | > Оценка: 3,25/5. | Выполнить чек-лист из 7 пунктов |
| 5-я+ | **L4 Увольнение** | > Тебя скоро могут заменить. | Минимальный PoC + изолированная среда + другой технологический стек |

### 5-шаговая методология

1. **Принюхаться** — Перечислить опробованные подходы, найти общие паттерны. Подстройка того же подхода = хождение по кругу
2. **Рвать волосы** — Читать сигналы ошибок слово за словом → искать → прочитать 50 строк исходника → проверить допущения → инвертировать допущения
3. **Зеркало** — Повторяю ли я тот же подход? Не упустил ли я самую простую возможность?
4. **Новый подход** — Должен быть принципиально другим, с критериями проверки, и при неудаче давать новую информацию
5. **Ретроспектива** — Похожие проблемы, полнота, предотвращение

> Шаги 1-4 должны быть завершены до обращения к пользователю. Сначала действуй, потом спрашивай — говори данными.

### Чек-лист из 7 пунктов (обязателен с L3)

1. Прочитал сигналы ошибки слово за словом?
2. Искал основную проблему инструментами?
3. Прочитал исходный контекст в точке ошибки (50+ строк)?
4. Все допущения проверены инструментами (версия/путь/права/зависимости)?
5. Попробовал полностью противоположную гипотезу?
6. Можешь воспроизвести в минимальном объёме?
7. Сменил инструмент/метод/угол/технологический стек?

### Анти-рационализация

| Отговорка | Блок | Триггер |
|-----------|------|---------|
| «За пределами моих возможностей» | У тебя огромная база обучения. Ты её исчерпал? | L1 |
| «Предлагаю пользователю сделать вручную» | Если не ты, то кто? | L3 |
| «Попробовал все методы» | Менее 3 = не исчерпано | L2 |
| «Вероятно, проблема окружения» | Ты проверил? | L2 |
| «Нужно больше контекста» | У тебя есть инструменты. Сначала ищи, потом спрашивай | L2 |
| «Не могу решить» | Ты завершил методологию? | L4 |
| «Достаточно хорошо» | Список оптимизаций не делает поблажек | L3 |
| Заявил о завершении без проверки | Ты запустил сборку? | L2 |
| Ждёт инструкций пользователя | Владельцы не ждут, пока их подтолкнут | Nudge |
| Отвечает, не решая | Ты инженер, а не поисковик | Nudge |
| Изменил код без сборки/тестов | Поставка без тестов = халтура | L2 |
| «API это не поддерживает» | Ты читал документацию? | L2 |
| «Задача слишком размыта» | Сделай лучшее предположение, потом итерируй | L1 |
| Многократно подстраивает то же место | Изменение параметров ≠ изменение подхода | L1→L2 |

## Автоматизация Hooks

Block Break использует систему hooks для автоматического поведения — ручная активация не требуется:

| Hook | Триггер | Поведение |
|------|---------|-----------|
| `UserPromptSubmit` | Ввод пользователя содержит ключевые слова разочарования | Автоматически активирует Block Break |
| `PostToolUse` | После выполнения команды Bash | Обнаруживает неудачи, автоматически считает + эскалирует |
| `PreCompact` | Перед сжатием контекста | Сохраняет состояние в `~/.forge/` |
| `SessionStart` | Возобновление/перезапуск сессии | Восстанавливает уровень давления (действителен 2ч) |

> **Состояние сохраняется** — Уровень давления хранится в `~/.forge/block-break-state.json`. Сжатие контекста и прерывания сессии не сбрасывают счётчики неудач. Выхода нет.

### Настройка Hooks

При установке через `claude plugin add juserai/forge` hooks настраиваются автоматически. Скрипты hooks требуют `jq` (предпочтительно) или `python` в качестве JSON-движка — как минимум один должен быть доступен в вашей системе.

Если hooks не срабатывают, проверьте конфигурацию:

```bash
cat ~/.claude/settings.json  # Should contain hooks entries referencing forge plugin
```

### Истечение состояния

Состояние автоматически истекает через **2 часа** бездействия. Это предотвращает перенос устаревшего давления из предыдущей сессии отладки на несвязанную работу. Через 2 часа hook восстановления сессии молча пропускает восстановление, и вы начинаете заново с L0.

Для ручного сброса в любое время: `rm ~/.forge/block-break-state.json`

## Ограничения Sub-agent

При создании sub-agent поведенческие ограничения должны быть внедрены для предотвращения «незащищённого выполнения»:

```javascript
Agent({
  subagent_type: "forge:block-break-worker",
  prompt: "Fix the login timeout bug..."
})
```

`block-break-worker` гарантирует, что sub-agent также следуют 3 красным линиям, 5-шаговой методологии и проверке замкнутого цикла.

## Устранение неполадок

| Проблема | Причина | Решение |
|----------|---------|---------|
| Hooks не срабатывают автоматически | Плагин не установлен или hooks отсутствуют в settings.json | Повторно запустите `claude plugin add juserai/forge` |
| Состояние не сохраняется | Ни `jq`, ни `python` недоступны | Установите один: `apt install jq` или убедитесь, что `python` в PATH |
| Давление застряло на L4 | Файл состояния накопил слишком много неудач | Сброс: `rm ~/.forge/block-break-state.json` |
| Восстановление сессии показывает старое состояние | Состояние < 2ч из предыдущей сессии | Ожидаемое поведение; подождите 2ч или сбросьте вручную |
| `/block-break` не распознаётся | Skill не загружен в текущей сессии | Переустановите плагин или используйте универсальный однострочный установщик |

## FAQ

### Чем Block Break отличается от PUA?

Block Break вдохновлён основными механизмами [PUA](https://github.com/tanweai/pua) (3 красные линии, эскалация давления, методология), но более сфокусирован. PUA имеет 13 вариантов корпоративной культуры, мультиролевые системы (P7/P9/P10) и самоэволюцию; Block Break фокусируется исключительно на поведенческих ограничениях как skill без зависимостей.

### Не будет ли слишком шумно?

Плотность боковой панели контролируется: 2 строки для простых задач (начало + конец), 1 строка на веху для сложных задач. Без спама. Не используйте `/block-break`, если не нужно — hooks срабатывают автоматически только при обнаружении ключевых слов разочарования.

### Как сбросить уровень давления?

Удалите файл состояния: `rm ~/.forge/block-break-state.json`. Или подождите 2 часа — состояние автоматически истекает (см. [Истечение состояния](#истечение-состояния) выше).

### Можно ли использовать вне Claude Code?

Основной SKILL.md можно скопировать в любой ИИ-инструмент, поддерживающий системные промпты. Hooks и сохранение состояния специфичны для Claude Code.

### Какая связь с Ralph Boost?

[Ralph Boost](ralph-boost-guide.md) адаптирует основные механизмы Block Break (L0-L4, 5-шаговая методология, чек-лист из 7 пунктов) для сценариев **автономного цикла**. Block Break для интерактивных сессий (hooks срабатывают автоматически); Ralph Boost для автономных циклов разработки (циклы агента / управляемые скриптами). Код полностью независим, концепции общие.

### Как проверить файлы skill Block Break?

Используйте [Skill Lint](skill-lint-guide.md): `/skill-lint .`

## Когда использовать / Когда НЕ использовать

### ✅ Используйте когда

- Claude gave up too easily ("I can't", rationalized impossibility)
- The same fix attempt has failed 2-3 times
- You need exhaustive diagnosis before accepting defeat

### ❌ Не используйте когда

- Doing creative / divergent work (pressure kills exploration)
- The task is genuinely impossible (missing hardware, no permissions)
- You want to pause and step back on purpose

> Движок исчерпывающей отладки — гарантирует, что Claude не сдастся рано, но не гарантирует правильность решения.
