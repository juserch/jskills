# Руководство пользователя Insight Fuse

> Систематический многоисточниковый исследовательский движок — От темы до профессионального аналитического отчета

## Быстрый старт

```bash
# Полное исследование (5 этапов, с ручными контрольными точками)
/insight-fuse AI Agent риски безопасности

# Быстрое сканирование (только Stage 1)
/insight-fuse --depth quick квантовые вычисления

# Использование конкретного шаблона
/insight-fuse --template technology WebAssembly

# Глубокое исследование с пользовательскими перспективами
/insight-fuse --depth deep --perspectives optimist,pessimist,pragmatist коммерциализация автономного вождения
```

## Параметры

| Параметр | Описание | Пример |
|----------|----------|--------|
| `topic` | Тема исследования (обязательно) | `AI Agent риски безопасности` |
| `--depth` | Глубина исследования | `quick` / `standard` / `deep` / `full` |
| `--template` | Шаблон отчета | `technology` / `market` / `competitive` |
| `--perspectives` | Список перспектив | `optimist,pessimist,pragmatist` |

## Режимы глубины

### quick — Быстрое сканирование
Выполняет Stage 1. 3+ поисковых запроса, 5+ источников, выводит краткий отчет. Подходит для быстрого ознакомления с темой.

### standard — Стандартное исследование
Выполняет Stage 1 + 3. Автоматически определяет подвопросы, параллельное исследование, полный охват. Без ручного взаимодействия.

### deep — Глубокое исследование
Выполняет Stage 1 + 3 + 5. На основе стандартного исследования проводит углубленный анализ всех подвопросов с 3 перспектив. Без ручного взаимодействия.

### full (по умолчанию) — Полный конвейер
Выполняет все 5 этапов. Stage 2 и Stage 4 — ручные контрольные точки для контроля направления.

## Шаблоны отчетов

### Встроенные шаблоны

- **technology** — Технологическое исследование: архитектура, сравнение, экосистема, тренды
- **market** — Исследование рынка: объем, конкуренция, пользователи, прогнозы
- **competitive** — Конкурентный анализ: матрица функций, SWOT, ценообразование

### Пользовательские шаблоны

1. Скопируйте `templates/custom-example.md` как `templates/your-name.md`
2. Измените структуру глав
3. Сохраните заполнители `{topic}` и `{date}`
4. Последняя глава должна быть «Справочные источники»
5. Активируйте с помощью `--template your-name`

### Режим без шаблона

Если `--template` не указан, агент адаптивно генерирует структуру отчета на основе содержания исследования.

## Многоперспективный анализ

### Перспективы по умолчанию

| Перспектива | Роль | Модель |
|-------------|------|--------|
| Generalist | Широкий охват, мейнстримный консенсус | Sonnet |
| Critic | Критическая проверка, поиск контраргументов | Opus |
| Specialist | Техническая глубина, первичные источники | Sonnet |

### Альтернативные наборы перспектив

| Сценарий | Перспективы |
|----------|------------|
| Прогнозирование трендов | `--perspectives optimist,pessimist,pragmatist` |
| Исследование продукта | `--perspectives user,developer,business` |
| Исследование политик | `--perspectives domestic,international,regulatory` |

### Пользовательские перспективы

Создайте `agents/insight-{name}.md`, используя структуру существующих файлов агентов в качестве образца.

## Обеспечение качества

Каждый отчет автоматически проверяется:
- Минимум 2 независимых источника на главу
- Отсутствие висячих ссылок
- Доля одного источника не превышает 40%
- Все сравнительные утверждения подкреплены данными

## Когда использовать / Когда НЕ использовать

### ✅ Используйте когда

- Writing a professional research report from multiple sources
- You want configurable depth (scan → deep-dive)
- Multi-perspective crossing of new information

### ❌ Не используйте когда

- Quick factual lookup (overkill; use `claim-ground` / WebSearch)
- Single-source deep reading
- Tasks requiring primary research (interviews, field work)

> Конвейер desk research — превращает многоисточниковый синтез в настраиваемый процесс, не делает первичных исследований.

## Отличие от council-fuse

| | insight-fuse | council-fuse |
|---|---|---|
| **Назначение** | Активное исследование + генерация отчетов | Многоперспективное обсуждение известной информации |
| **Источник информации** | Сбор через WebSearch/WebFetch | Вопросы от пользователя |
| **Вывод** | Полный аналитический отчет | Синтезированный ответ |
| **Этапы** | 5 этапов поэтапно | 3 этапа (созыв → оценка → синтез) |

Оба инструмента можно комбинировать: сначала используйте insight-fuse для исследования и сбора информации, затем council-fuse для углубленного обсуждения ключевых решений.
