# insight-fuse — 能力边界

## 定位

- **分类**：crucible（多源融合，输出专业调研报告）
- **触发**：`/insight-fuse <topic>` + 可选参数（`--depth` / `--template` / `--perspectives`）
- **本质**：5 阶段渐进式流水线（扫描 → 对齐 → 调研 → 审阅 → 深挖）+ 可选多视角分析

## ✅ 能解决的问题

| 问题类型 | 机制 | 典型案例 |
|---------|------|---------|
| 系统化多源调研 | 5 阶段 + 质量标准（多源强制） | 写一份 "AI Agent 安全风险" 报告 |
| 深度可配置 | quick / standard / deep / full | 同一主题要快速扫描还是深度挖 |
| 多视角交叉 | 通才 / 批评者 / 专家 3 视角 | 既要广度也要反驳 |
| 报告模板化 | technology / market / competitive + 自定义 | 不同目的用不同结构 |

## ❌ 不能解决的问题

### 设计边界内（故意不管）

| 问题 | 为什么不管 |
|------|-----------|
| 实时数据 | WebSearch / WebFetch 源可能有缓存 / 延迟 |
| 无网络覆盖的 niche 话题 | 搜不到就没法综合 |

### 超出设计能力（接不住）

| 问题 | 为什么接不住 |
|------|-------------|
| 主观质量判断 | "哪个设计更好"——没客观源可引 |
| 需要真人访谈的 primary research | 只能做 desk research，不采访 |
| 付费墙 / gated 源 | 看不到内容，只能看到标题 |

## 🚫 不应使用场景

- **快速事实查询**：过度工程；用 claim-ground 或 WebSearch 即可
- **单源深度阅读**：insight-fuse 的价值在多源交叉，单源用不着流水线
- **需要专家采访的任务**：AI 做不了人际互动

## 一句话定位

> insight-fuse 是**desk research 的流水线**——它把"看 20 个源 + 写一份报告"变成可配置流程，但不做 primary research（采访、现场调查、付费墙），也不保证源本身的时效。
