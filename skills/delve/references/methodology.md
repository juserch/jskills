# 五步方法论（卡壳时强制执行）

每次失败或卡壳后按以下 5 步执行。代码、研究、写作、规划都适用。

## 1. 闻味道

列出所有尝试过的方案，找共同模式。同一思路微调 = 原地打转。

## 2. 揪头发

按序执行（跳过任何一个 = 3.25）：

1. **逐字读失败信号** — 90% 的答案你直接忽略了
2. **主动搜索** — 报错原文 / 官方文档 / 多角度关键词
3. **读原始材料** — 源码上下文 50 行，不是摘要
4. **验证前置假设** — 版本、路径、权限、依赖——用工具确认，不要猜
5. **反转假设** — 一直假设"问题在 A"→ 现在假设"问题不在 A"

## 3. 照镜子

- 是否在重复同一思路？
- 是否该搜索却没搜？
- 是否忽略了最简单的可能？

## 4. 执行新方案

必须和之前**本质不同**：
- 有明确验证标准
- 失败时能产生新信息
- 换参数 / 换函数名不叫换方案

## 5. 复盘

- 检查同类问题（冰山法则）
- 修复完整性
- 预防措施

**规则**：步骤 1-4 完成前不允许向用户提问——先做后问，用数据说话。
